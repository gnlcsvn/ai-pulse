## Topics Discussed
- **Human bottleneck to AGI**: The key constraint is not model capability but humans' limited ability to prompt and validate AI output -- people use AI 30 times/day but could benefit tens of thousands of times [▶ 5:35](https://www.youtube.com/watch?v=S1rQngjpUdI&t=335s)
- **Three phases of AI agent evolution**: Phase 1: agents work well for coding. Phase 2: realize all agents are coding agents, build flexible tools for tinkerers. Phase 3: productize specific features that work out of the box. Emberikos says they will "speed run" all three phases in coming months [▶ 10:44](https://www.youtube.com/watch?v=S1rQngjpUdI&t=644s)
- **Coding is being automated**: Agrees with Elon that coding is one of the first domains where LLMs excel, but pushes back on the framing -- "what does it mean for coding to be automated?" is a nuanced question since software engineering involves far more than writing code [▶ 2:15](https://www.youtube.com/watch?v=S1rQngjpUdI&t=135s)
- **Codex app design philosophy**: Deliberately omitted text editing to emphasize delegation over pairing; built with affordances for managing multiple agents, delegating tasks, and reviewing changes rather than writing code [▶ 19:37](https://www.youtube.com/watch?v=S1rQngjpUdI&t=1177s)
- **OpenAI's Atlas browser**: Building a browser to enable safe agentic browsing for enterprise, providing an alternative path to AI adoption that avoids the need for full FDE-built integrations [▶ 14:07](https://www.youtube.com/watch?v=S1rQngjpUdI&t=847s)
- **Open standards strategy (agents.md, skills)**: OpenAI pushes vendor-neutral conventions like agents.md (adopted by every agent except Claude) and a neutral agents/ folder for skills, deliberately avoiding vendor lock-in [▶ 23:06](https://www.youtube.com/watch?v=S1rQngjpUdI&t=1386s)

- **Code review in the AI era**: Plan review is becoming more important than code review as the paradigm shifts to delegation; Codex is trained explicitly for high-signal code review with few false positives [▶ 20:07](https://www.youtube.com/watch?v=S1rQngjpUdI&t=1207s)
- **Competitive dynamics**: Does not believe the market will be monopolistic; sees competition as beneficial because OpenAI learns when competitors improve; serving models to competitors is aligned with the mission [▶ 26:01](https://www.youtube.com/watch?v=S1rQngjpUdI&t=1561s)

## Summary
Alexander Emberikos, product lead for Codex at OpenAI, joins Harry Stebbings for a wide-ranging conversation about the transformation of software engineering, the design philosophy behind the Codex platform, and OpenAI's broader strategy for distributing AI intelligence. Emberikos argues that the key bottleneck to AI's impact is not model capability but the human interface -- people currently use AI about 30 times per day when it could theoretically help tens of thousands of times, limited primarily by the effort of typing prompts and the imagination to recognize where AI could assist.

The conversation reveals that a fundamental shift has already occurred within OpenAI itself. With the release of GPT 5.2 Codex in December 2025, engineers largely stopped opening traditional code editors and transitioned to a "delegation" model where they plan and spec work collaboratively with AI agents, then let the agents execute. The vast majority of code at OpenAI is now written by AI, and nearly all code is automatically reviewed by Codex upon being pushed to repositories. Emberikos describes three phases of AI agent evolution: first making agents excellent at coding, then recognizing that all agents are fundamentally coding agents (since code is the best way for agents to interact with computers), and finally productizing specific features for broad adoption.

Strategically, Emberikos describes an approach that is counterintuitive to traditional venture thinking. OpenAI deliberately serves its models to competitors, pushes for open standards like agents.md, and keeps the Codex core harness open source. The rationale is that their mission is the "distribution of intelligence" and that competition helps them learn faster. The company is also building Atlas, its own browser, to enable safe agentic browsing for enterprises -- solving the data security and access control challenges that currently block large organizations from fully adopting AI tools. On the question of what defines winning, Emberikos points to compute advantage and having the best models at the company level, while emphasizing product execution and building for individual users at the product level.

The interview also touches on the diminishing role of traditional product managers, the problem of AI "slop" in open source contributions, and the importance of plan review (vs. code review) in an age of agent delegation. Emberikos emphasizes that the most exciting future is one where AI makes everyone feel like a "superhuman" without requiring expertise in prompting or tool usage.
